Task Instructions

Please read the instructions carefully before starting the task.

In the next part of the study, you will play the role of an engineer investigating an unfamiliar machine with four nodes: N1, N2, and N3 are switches you can turn ON or OFF, and N4 is an outcome light controlled by a hidden rule that depends on some combination of N1, N2, and N3.

N1, N2, and N3 are switches you control. N4 is the outcome light — its hidden rule is what you'll be trying to discover.

There will be 15 trials. On each trial, the machine has a new hidden rule. You can test the machine as many times as you like — set the switches however you like, then press Test to see whether N4 turns ON — before typing out, in your own words, what you think the hidden rule is. There is no right or wrong way to explore, just do what feels natural to you.

Examples of rules you may encounter

1. Single-feature concept

IF [node] = [state], THEN N4 turns ON.

Example: IF N1 = ON, THEN N4 = ON. (N2 and N3 would not matter.)

2. Conjunctive / disjunctive concept

IF [node A] = [state] AND/OR [node B] = [state], THEN N4 turns ON.

Example (conjunctive, ON): IF N1 = ON AND N2 = ON, THEN N4 = ON.

Example (conjunctive, OFF): IF N1 = OFF AND N2 = OFF, THEN N4 = ON. (Being ON does not help.)

Example (disjunctive): IF N1 = ON OR N2 = ON, THEN N4 = ON.

Use testing to figure out which rule the machine is following on each trial.
