Task Instructions

You will play the role of an engineer investigating an unfamiliar machine. The machine has four nodes: N1, N2, and N3 are switches you can turn ON or OFF. N4 is an outcome light controlled by a hidden rule that depends on some combination of N1, N2, and N3.

There will be 15 trials. On each trial the machine has a new hidden rule. Your job is to run tests — set the switches however you like, then press Test to see whether N4 turns ON — for as long as you like, until you feel ready to describe the rule in your own words.

Kinds of rules you may encounter

Each trial's hidden rule will be one of the following three kinds:

1. Single-feature concept

IF [node] = [state], THEN N4 turns ON.

Example: IF N1 = ON, THEN N4 = ON. (N2 and N3 would not matter.)

2. Conjunctive / disjunctive concept

IF [node A] = [state] AND/OR [node B] = [state], THEN N4 turns ON.

Example (conjunctive): IF N1 = ON AND N2 = ON, THEN N4 = ON.

Example (disjunctive): IF N1 = ON OR N2 = ON, THEN N4 = ON.

3. Relational ("Match") concept

IF [node A] and [node B] are in the SAME state (both ON or both OFF), THEN N4 turns ON.

Example: IF N1 and N2 match (both ON or both OFF), THEN N4 = ON.

Keep these three possibilities in mind as you test the machine on each trial.
